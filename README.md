## 🌌 What Is Syscall Detective?

<table>
<tr>
<td valign="top" width="58%">

**Syscall Detective** is an elite, AI-powered forensics platform that transforms raw Linux `strace` output into a beautiful, actionable intelligence dashboard — **no source code required**.

Every program running on Linux — a simple `ls`, a web server, or a stealthy malware payload — **cannot touch hardware directly**. Every file open, network connection, and memory allocation must flow through a strict gateway: **the Linux Kernel via System Calls (Syscalls)**.

Syscall Detective places a transparent lens at that exact gateway. It captures the conversation in real time, then renders it as:

- 🕸️ Interactive node-based **Call Graphs**
- 📊 Color-coded **Syscall Histograms**
- 🧠 Plain-English **AI Forensic Narratives**
- 🎯 Weighted **Risk / Weirdness Scores**
- 📧 One-click **Emailed PDF Reports**

> **Drop a binary. Get a forensic report. No terminal expertise needed.**

## 🧮 Weirdness Score™

The Weirdness Score is the crown jewel of Syscall Detective — **zero manual analysis required**. The engine iterates over every captured syscall, accumulates weighted penalties, then visualizes a final 0–100 risk rating.

### ⚖️ Heuristic Weight Table

| Syscall | Threat Rationale | Risk Weight |
|:---|:---|:---:|
| `ptrace` | Attaches a debugger to a running process — primary vector for code injection attacks | 🔴 **+25 pts** |
| `execve` | Spawns a secondary executable or hidden shell payload | 🟠 **+10 pts** |
| `connect` · `bind` | Unexpected outbound network socket — potential C2 beacon or data exfiltration | 🟡 **+8 pts** |
| `mprotect` | Marks memory regions as executable — classic shellcode staging pattern | 🟡 **+5 pts** |
| `unlink` | Deletes files — potential self-cleanup / track-covering behavior | 🟢 **+3 pts** |

### 📶 Score Bands

```
  0 ─────────── 20 ──────────────────── 49 ──────────────────────────── 100
  │             │                       │                                 │
  │  ✅  SAFE   │    ⚠️  SUSPICIOUS      │     🚨  CRITICAL                │
  │             │                       │                                 │
  │  Standard   │  Warrants deeper      │  Dashboard turns red.           │
  │  execution  │  manual review        │  Alerts flash. AI confirms.     │
  │  profile    │  of flagged calls     │  Isolate the binary now.        │
```

## ⚙️ Environment Configuration

All runtime secrets are stored exclusively in `.env.local` — a file that is **never committed to version control**. The repository ships a `.env.example` template with placeholder descriptions and no real values.

### Required Variables

| Variable | Purpose | Where to Obtain |
|:---|:---|:---|
| `RESEND_API_KEY` | Signs and dispatches transactional email reports | [resend.com](https://resend.com) → API Keys |
| `CONTACT_EMAIL` | Destination for in-app contact form submissions | Your own email address |
| `NOTIFICATION_EMAIL` | Destination for exported AI analysis reports | Your own email address |

### Setup Steps

**1 · Obtain an Email API Key**

- Create a free account at **[resend.com](https://resend.com)** — free tier includes 3,000 emails/month
- Navigate to **API Keys → Create API Key** and copy the generated string

**2 · Populate `.env.local`**

- Copy the template: `cp .env.example .env.local`
- Open `.env.local` in any editor and replace each placeholder with real values
- Save — the file is already listed in `.gitignore` and will never be pushed

**3 · Verify the Integration**

- Run `npm run dev`, then submit the contact form or export a report
- Confirm the email arrives in your inbox within seconds

> 🛡️ **Privacy by Design** — Every variable is sealed server-side. The UI always renders `Host` / `Admin` where a real address would otherwise appear. Deployed visitors and source-code inspectors can never see your credentials.

## ❓ FAQ

<details>
<summary><b>Do I need the program's source code to analyze it?</b></summary>
<br/>
No — that is the core power of syscall tracing. Every compiled ELF binary must communicate with the Linux kernel to do anything meaningful. Syscall Detective monitors that exact conversation, completely bypassing the need for source code or debug symbols.
</details>

<details>
<summary><b>Can I use this on Windows or macOS?</b></summary>
<br/>
The Syscall Detective web dashboard runs in any modern browser on any operating system. The <code>strace</code> utility for generating raw trace logs is Linux-only. Windows and macOS users can still access every feature via the pre-configured program library and drag-and-drop <code>.log</code> file upload.
</details>

<details>
<summary><b>Is it safe to analyze real malware samples?</b></summary>
<br/>
The Docker sandbox is isolated and non-persistent — containers are destroyed after each run. For advanced red-team or unknown payloads, generate the <code>.log</code> file inside a dedicated air-gapped VM, then upload only that file to Syscall Detective. Never execute unknown binaries on production hardware.
</details>

<details>
<summary><b>What exactly triggers a Critical (50+) Weirdness Score?</b></summary>
<br/>
A score of 50+ activates Critical Alert mode: the entire dashboard turns red, warning banners flash, and the AI narrative confirms the threat. Syscall Detective is a forensic analysis tool — it does not block execution. Always combine its output with your own judgment and further manual investigation.
</details>

<details>
<summary><b>Can I self-host Syscall Detective?</b></summary>
<br/>
Yes. The project runs on any Node.js 18+ environment. Clone the repository, configure your <code>.env.local</code> variables, run <code>npm run build && npm run start</code>, and point a reverse proxy (nginx / Caddy) at port 3000. Vercel deployment is the recommended zero-config path.
</details>
